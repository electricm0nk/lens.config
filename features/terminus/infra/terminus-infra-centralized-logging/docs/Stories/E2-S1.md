# E2-S1 — Alloy DaemonSet Deployment: Helm Values + ArgoCD App

## Story

**As an** operator,  
**I want** Grafana Alloy deployed as a DaemonSet on all k3s nodes,  
**so that** pod logs from all workloads are continuously collected and shipped to prod Loki.

## Context

Alloy is the cluster-side log collector. It reads `/var/log/pods/` and `/var/log/journal` on each node and ships to prod Loki at `loki.monitoring.svc.cluster.local:3100`. Single DaemonSet ships to prod only — no `alloy-dev`.

**Pre-req:** E1-S2 complete (prod Loki healthy and accepting pushes).

**Open item OI-2:** Before deploying, verify Alloy DaemonSet compatibility with containerd 2.0.4 (check Alloy release notes for containerd 2.x support). If incompatible, pin containerd version on nodes.

**Relevant architecture sections:**
- `architecture.md` §Phase 2 — Alloy DaemonSet
- `architecture.md` §Component Decisions §2 — Grafana Alloy DaemonSet

## Key Implementation Points

- DaemonSet toleration: `node-role.kubernetes.io/control-plane: NoSchedule`
- Label strategy: `cluster=terminus`, `namespace`, `app` (from `app.kubernetes.io/name`), `pod`, `container`, `node`, `env`
- No `log_level` label at collection time (cardinality risk)
- ServiceAccount: `get`/`list`/`watch` on `pods`, `namespaces` only
