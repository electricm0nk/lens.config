# E6-S1 — Alloy VM Ansible Playbook: Vault and Postgres VM Log Shipping

## Story

**As an** operator,  
**I want** Alloy installed on Vault VM and Postgres VM as systemd services,  
**so that** Vault audit logs and Postgres query logs are shipped to prod Loki via `loki-push.trantor.internal`.

## Context

Delivery via Ansible playbook run from SemaphoreUI (consistent with existing VM management). VM Alloy ships logs to `http://loki-push.trantor.internal/loki/api/v1/push` — the Traefik ingress created in E1-S2. The Alloy config is per-VM (different log sources).

**Pre-reqs:** E1-S2 complete (prod Loki healthy + `loki-push.trantor.internal` ingress live).

## Files to Create

- `platforms/vm/alloy/config.alloy.vault.j2` — Vault VM Alloy config template
- `platforms/vm/alloy/config.alloy.postgres.j2` — Postgres VM Alloy config template
- `platforms/vm/alloy/deploy-alloy-vm.yaml` — Ansible playbook

## Key Config Snippets

### Vault VM Config (`config.alloy.vault.j2`)

```alloy
local.file_match "vault_audit" {
  path_targets = [{"__path__" = "/var/log/vault/audit.log"}]
}

loki.source.file "vault_audit" {
  targets    = local.file_match.vault_audit.targets
  forward_to = [loki.write.default.receiver]
}

loki.write "default" {
  endpoint {
    url = "http://loki-push.trantor.internal/loki/api/v1/push"
  }
  external_labels = {
    cluster  = "terminus",
    service  = "vault",
    log_type = "audit",
  }
}
```

### Ansible Playbook Highlights

```yaml
# deploy-alloy-vm.yaml
- name: Install Alloy on VM
  hosts: "{{ target_host }}"
  vars:
    alloy_version: "1.0.0"  # pin version — no auto-update
    alloy_position_path: /var/lib/alloy/positions.yaml
  tasks:
    - name: Install Alloy binary
      ...
    - name: Write Alloy config from template
      template:
        src: "config.alloy.{{ vm_type }}.j2"
        dest: /etc/alloy/config.alloy
    - name: Enable alloy systemd service
      systemd:
        name: alloy
        enabled: true
        state: started
```
