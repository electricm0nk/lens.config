# E2-S2 — Alloy Log Stream Validation

## Story

**As an** operator,  
**I want** to verify log streams are flowing correctly from all namespaces to Loki,  
**so that** I'm confident no workloads are missing from the log pipeline.

## Context

Validation story — no new files to create. Run LogQL queries to confirm coverage, label strategy, and cardinality.

**Pre-req:** E2-S1 complete (Alloy DaemonSet running).

## Key Validation Queries (run in Grafana Explore or Loki API)

```logql
# Cluster-wide coverage check
{cluster="terminus"} | count by (namespace)

# Specific namespace
{cluster="terminus", namespace="argocd"} | tail 20

# Dev namespace (env label)
{cluster="terminus", env="dev"} | tail 10

# Vault audit logs (if E6 not done yet, should be empty)
{service="vault"} | tail 5
```

## Acceptance Criteria

- [ ] LogQL `{cluster="terminus", namespace="argocd"}` returns recent logs
- [ ] LogQL `{cluster="terminus", namespace="monitoring"}` returns Prometheus/Grafana logs
- [ ] LogQL `{cluster="terminus", env="dev"}` returns dev namespace logs
- [ ] Label cardinality: 6 labels per stream — no unexpected high-cardinality labels
- [ ] No Alloy push error logs in `kubectl logs -n monitoring ds/alloy`
- [ ] No `alloy` pod evictions or restarts due to resource limits
