# E1-S2 — Loki Prod Deployment: Helm Values + ArgoCD App

## Story

**As an** operator,  
**I want** Loki deployed in the `monitoring` namespace with MinIO in-cluster S3 storage,  
**so that** cluster logs can be ingested and retained for 30 days in durable block storage separate from the Loki pod.

## Context

Loki is the central log backend. This story creates all GitOps artifacts to deploy the `loki` ArgoCD multi-source app following the `prometheus.yaml` pattern. Prod uses MinIO in-cluster S3 backend with `terminus-loki-prod` bucket, 30d retention, `compactor.retention_enabled: true`.

This story also includes creating the `loki-push.trantor.internal` Traefik ingress, required by E6 (VM log shipping).

## Implementation Tasks

### Task 1: Create Loki Helm Values (prod)

Create `platforms/k3s/helm/loki/values.yaml`:

```yaml
loki:
  commonConfig:
    replication_factor: 1
  auth_enabled: false   # single-tenant mode

  storage:
    type: s3
    s3:
      endpoint: http://minio.monitoring.svc.cluster.local:9000
      bucketnames: terminus-loki-prod
      region: us-east-1                # required field; MinIO ignores the value
      access_key_id: ${LOKI_S3_ACCESS_KEY}
      secret_access_key: ${LOKI_S3_SECRET_KEY}
      insecure: true                   # MinIO in-cluster HTTP — no TLS needed internally
      s3forcepathstyle: true           # required for MinIO path-style access

  schemaConfig:
    configs:
      - from: "2026-04-22"
        store: tsdb
        object_store: s3
        schema: v13
        index:
          prefix: index_
          period: 24h

  compactor:
    retention_enabled: true           # REQUIRED — without this retention_period is silently ignored

  limits_config:
    retention_period: 720h            # 30 days
    ingestion_rate_mb: 16
    ingestion_burst_size_mb: 32
    max_streams_per_user: 0           # unlimited in single-tenant mode

  # Ruler is configured in E5-S1 — stub here so it doesn't error on missing config
  ruler:
    enabled: false                    # enabled in E5-S1

singleBinary:
  replicas: 1
  persistence:
    enabled: false                    # prod uses S3 — no PVC needed for storage
  resources:
    requests:
      cpu: 100m
      memory: 256Mi
    limits:
      cpu: 500m
      memory: 512Mi

# ExternalSecret values are injected at runtime via Kubernetes Secret envFrom
# loki-s3-creds Secret keys: access_key_id, secret_access_key
# Injected via `extraEnvFrom` below
extraEnvFrom:
  - secretRef:
      name: loki-s3-creds
```

**Note on credentials injection:** The `${LOKI_S3_ACCESS_KEY}` and `${LOKI_S3_SECRET_KEY}` placeholders require the Loki Helm chart to support environment variable substitution, OR use a Helm post-renderer, OR inject via `env` from the Secret. Verify the `grafana/loki` chart version's mechanism. If env substitution is not supported, use `loki.storage.s3.accessKeyId` referencing the Secret via `valueFrom.secretKeyRef` in the ArgoCD app's Helm parameters.

### Task 2: Create ArgoCD App Manifest

Create `platforms/k3s/argocd/apps/loki.yaml` following the multi-source pattern from `prometheus.yaml`:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: loki
  namespace: argocd
  annotations:
    argocd.argoproj.io/sync-wave: "3"
spec:
  project: terminus-infra
  sources:
    - repoURL: https://grafana.github.io/helm-charts
      chart: loki
      targetRevision: "6.*"
      helm:
        valueFiles:
          - $values/platforms/k3s/helm/loki/values.yaml
    - repoURL: https://github.com/electricm0nk/terminus.infra.git
      targetRevision: HEAD
      ref: values
  destination:
    server: https://kubernetes.default.svc
    namespace: monitoring
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=false   # monitoring namespace already exists
```

### Task 3: Create Loki Push Traefik Ingress

Create `platforms/k3s/k8s/monitoring/loki-push-ingress.yaml`:

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: loki-push
  namespace: monitoring
  annotations:
    kubernetes.io/ingress.class: traefik
    cert-manager.io/cluster-issuer: vault-issuer
spec:
  tls:
    - hosts:
        - loki-push.trantor.internal
      secretName: loki-push-tls
  rules:
    - host: loki-push.trantor.internal
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: loki
                port:
                  number: 3100
```

**Note:** Verify service name `loki` and port `3100` after Helm deploy. Adjust to actual service name from `kubectl get svc -n monitoring`.

### Task 4: Apply and Verify

```bash
# ArgoCD picks up the new app manifest — no manual kubectl needed if repo is already watched
# Or force sync:
argocd app sync loki

# Watch pod come up
kubectl get pods -n monitoring -l app.kubernetes.io/name=loki -w

# Test push
kubectl run -n monitoring loki-test --image=curlimages/curl --restart=Never --rm -it -- \
  curl -XPOST -H "Content-Type: application/json" \
  -d '{"streams":[{"stream":{"job":"test"},"values":[["'$(date +%s%N)'","test log entry"]]}]}' \
  http://loki.monitoring.svc.cluster.local:3100/loki/api/v1/push

# Verify object in MinIO bucket (MinIO console via port-forward or mc CLI)
```
