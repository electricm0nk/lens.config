# E4-S1 — Loki Grafana Datasource: Sidecar ConfigMaps

## Story

**As a** platform engineer,  
**I want** Loki available as a Grafana datasource in both prod and dev Grafana instances,  
**so that** I can run LogQL queries from the Grafana Explore view.

## Context

Grafana already has a sidecar container watching for ConfigMaps with `grafana_datasource: "1"` label in the same namespace. Create two ConfigMaps following this existing pattern — no changes to `grafana/values.yaml` needed.

**Open item OI-3:** Verify `sidecar.dashboards.enabled: true` is set in Grafana values. If sidecar is not enabled, that explains why new ConfigMaps are not picked up.

**Pre-reqs:** E1-S2 (prod Loki), E1-S3 (dev Loki).

## Files to Create

- `platforms/k3s/k8s/monitoring/loki-datasource.yaml`
- `platforms/k3s/k8s/monitoring-dev/loki-datasource-dev.yaml`

## Key Spec

```yaml
# loki-datasource.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: loki-datasource
  namespace: monitoring
  labels:
    grafana_datasource: "1"
data:
  datasource.yaml: |
    apiVersion: 1
    datasources:
      - name: Loki
        type: loki
        access: proxy
        url: http://loki.monitoring.svc.cluster.local:3100
        isDefault: false
        jsonData:
          maxLines: 1000
```

## Acceptance Criteria

- [ ] `loki-datasource.yaml` created in `monitoring` namespace
- [ ] `loki-datasource-dev.yaml` created in `monitoring-dev` namespace
- [ ] Loki datasource visible in prod Grafana `Configuration > Data Sources`
- [ ] Loki datasource visible in dev Grafana `Configuration > Data Sources`
- [ ] Grafana sidecar auto-reloads (no manual Grafana restart needed)
