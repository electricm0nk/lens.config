# E1-S1 — MinIO Deploy + S3 Credentials: ArgoCD App + Vault + ExternalSecret

## Story

**As an** operator,  
**I want** MinIO deployed in-cluster as an S3-compatible object store with credentials in Vault,  
**so that** Loki has durable block storage that is separate from the Loki pod lifecycle.

## Context

Loki prod uses MinIO as its S3-compatible object storage backend. MinIO runs as a single-instance StatefulSet in the `monitoring` namespace backed by a 100Gi `local-path` PVC. Storage is node-local (accepted for homelab scale). MinIO is a Phase 0 prerequisite — Loki cannot start without it.

Credentials (MinIO root user/password) are stored in Vault KV and materialized via ESO as `loki-s3-creds` in `monitoring`. The bucket `terminus-loki-prod` is auto-created by MinIO at startup via `defaultBuckets`.

- `architecture.md` §Phase 0 — Prerequisites
- `architecture.md` §Component Decisions §1 (Storage section)

## Pre-Implementation Checklist

- [ ] `monitoring` namespace exists (pre-existing — prometheus deployed there)
- [ ] Vault is accessible and KV secrets engine at `secret/` is enabled
- [ ] Check `kubectl get clustersecretstore` — note the ESO ClusterSecretStore name
- [ ] Verify `local-path` StorageClass is available: `kubectl get sc`

## Implementation Tasks

### Task 1: Store MinIO Credentials in Vault

Generate strong credentials for MinIO root user:

```bash
vault kv put secret/terminus/loki/s3-creds \
  access_key_id=<MINIO_ROOT_USER> \
  secret_access_key=<MINIO_ROOT_PASSWORD>
```

MinIO enforces: `access_key_id` ≥ 3 chars, `secret_access_key` ≥ 8 chars.

Verify:
```bash
vault kv get secret/terminus/loki/s3-creds
```

### Task 2: Create ExternalSecret for MinIO Credentials

Create `platforms/k3s/k8s/monitoring/loki-s3-creds-es.yaml`:

```yaml
apiVersion: external-secrets.io/v1beta1
kind: ExternalSecret
metadata:
  name: loki-s3-creds
  namespace: monitoring
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: vault-backend        # adjust to match kubectl get clustersecretstore
    kind: ClusterSecretStore
  target:
    name: loki-s3-creds
    creationPolicy: Owner
  data:
    - secretKey: access_key_id
      remoteRef:
        key: secret/terminus/loki/s3-creds
        property: access_key_id
    - secretKey: secret_access_key
      remoteRef:
        key: secret/terminus/loki/s3-creds
        property: secret_access_key
```

Apply and verify before proceeding to Task 3:
```bash
kubectl apply -f platforms/k3s/k8s/monitoring/loki-s3-creds-es.yaml
kubectl get externalsecret loki-s3-creds -n monitoring   # should show READY=True
kubectl get secret loki-s3-creds -n monitoring           # should exist
```

### Task 3: Create MinIO Helm Values

Create `platforms/k3s/helm/minio/values.yaml`:

```yaml
# bitnami/minio — single-instance, local-path PVC
mode: standalone

auth:
  existingSecret: loki-s3-creds
  rootUserSecretKey: access_key_id
  rootPasswordSecretKey: secret_access_key

defaultBuckets: "terminus-loki-prod"   # auto-creates bucket on startup

persistence:
  enabled: true
  size: 100Gi
  storageClass: local-path

resources:
  requests:
    cpu: 100m
    memory: 256Mi
  limits:
    cpu: 500m
    memory: 512Mi

service:
  type: ClusterIP
  ports:
    api: 9000
    console: 9001
```

**Note on `existingSecret`:** bitnami/minio reads root credentials from the referenced secret. The `loki-s3-creds` secret (materialized by ESO in Task 2) must exist before the MinIO pod starts. ArgoCD sync-wave ordering ensures this — ESO runs before wave 2.

### Task 4: Create ArgoCD App Manifest

Create `platforms/k3s/argocd/apps/minio.yaml`:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: minio
  namespace: argocd
  annotations:
    argocd.argoproj.io/sync-wave: "2"
spec:
  project: terminus-infra
  sources:
    - repoURL: https://charts.bitnami.com/bitnami
      chart: minio
      targetRevision: "14.*"
      helm:
        valueFiles:
          - $values/platforms/k3s/helm/minio/values.yaml
    - repoURL: https://github.com/electricm0nk/terminus.infra.git
      targetRevision: HEAD
      ref: values
  destination:
    server: https://kubernetes.default.svc
    namespace: monitoring
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=false
```

Wave 2 — deploys after ESO (wave 1) and before Loki (wave 3).

### Task 5: Verify

```bash
# Watch MinIO pod
kubectl get pods -n monitoring -l app.kubernetes.io/name=minio -w

# Check bucket auto-created (via MinIO console)
kubectl port-forward -n monitoring svc/minio 9001:9001
# Open http://localhost:9001 — log in with MinIO root credentials
# Confirm terminus-loki-prod bucket exists

# Or verify via curl S3 API
kubectl run -n monitoring s3-test --image=curlimages/curl --restart=Never --rm -it -- \
  curl -v http://minio.monitoring.svc.cluster.local:9000/terminus-loki-prod/
# Expected: 403 Forbidden (bucket exists but no auth provided) — not connection refused
```

## Notes

- MinIO enforces root password ≥ 8 chars; root user ≥ 3 chars
- `local-path` PVC is node-local — MinIO data survives pod restarts but not node loss (accepted for homelab scale)
- MinIO console port (9001) is not exposed externally; use `kubectl port-forward` for admin tasks
- S3 endpoint for Loki: `http://minio.monitoring.svc.cluster.local:9000` (HTTP — internal cluster traffic)
