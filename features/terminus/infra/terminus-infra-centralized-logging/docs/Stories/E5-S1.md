# E5-S1 — Loki Ruler Rules: Log-Based Alert Rules in values.yaml

## Story

**As an** operator,  
**I want** 5 log-based alert rules evaluated by the Loki ruler,  
**so that** I receive alerts for CrashLoops, OOMKills, Vault auth denials, Temporal panics, and ArgoCD errors.

## Context

Loki ruler is configured via `loki.rulesConfig` inline in `values.yaml`. No PrometheusRule CRDs. The ruler uses `alertmanager_url` to forward alerts — `alertmanager_url` must point to the Alertmanager service (already deployed by `prometheus` feature). Verify the service name before deploying.

```bash
kubectl get svc -n monitoring | grep alertmanager
```

Expected: `alertmanager-operated` on port 9093. If different, update the `alertmanager_url` in values.

**Pre-req:** E1-S2 complete (Loki running). E5-S2 (AlertmanagerConfig) can proceed in parallel.

## Files to Modify

- `platforms/k3s/helm/loki/values.yaml` (add ruler and rulesConfig sections)

## Key Config (from architecture.md)

```yaml
ruler:
  enabled: true
  storage:
    type: local
    local:
      directory: /rules
  rule_path: /rules/fake
  alertmanager_url: http://alertmanager-operated.monitoring.svc.cluster.local:9093

rulesConfig:
  terminus-platform.yaml: |
    groups:
      - name: terminus-platform
        rules:
          - alert: CrashLoopDetected
            expr: count_over_time({namespace=~".+"} |= "reason=CrashLoopBackOff" [5m]) > 0
            labels:
              severity: critical
          - alert: OOMKillDetected
            expr: count_over_time({namespace=~".+"} |= "OOMKilled" [5m]) > 0
            labels:
              severity: critical
          - alert: VaultAuditDeny
            expr: count_over_time({service="vault",log_type="audit"} | json | type="response" | error!="" [5m]) > 5
            labels:
              severity: warning
          - alert: TemporalWorkflowPanic
            expr: count_over_time({namespace="temporal"} |= "panic" [5m]) > 0
            labels:
              severity: critical
          - alert: ArgoCDSyncError
            expr: count_over_time({namespace="argocd"} |= "level=error" [5m]) > 10
            labels:
              severity: warning
```
