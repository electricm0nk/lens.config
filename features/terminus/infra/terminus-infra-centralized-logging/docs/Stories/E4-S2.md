# E4-S2 — Log Dashboards: Platform Log Search Dashboards

## Story

**As a** platform engineer,  
**I want** 5 Grafana dashboards for searching and exploring log data,  
**so that** I can quickly triage incidents using logs from ArgoCD, Temporal, fourdogs, platform services, and Vault.

## Context

5 dashboard ConfigMaps with `grafana_dashboard: "1"` label. Follow the same pattern as existing dashboard ConfigMaps in `platforms/k3s/k8s/prometheus-infra/`. Each uses Loki as datasource and LogQL queries.

**Pre-reqs:** E4-S1 (Loki datasource visible in Grafana), E2-S2 (log streams validated).

## Files to Create

- `platforms/k3s/k8s/prometheus-infra/dashboard-logs-argocd.yaml`
- `platforms/k3s/k8s/prometheus-infra/dashboard-logs-temporal.yaml`
- `platforms/k3s/k8s/prometheus-infra/dashboard-logs-fourdogs.yaml`
- `platforms/k3s/k8s/prometheus-infra/dashboard-logs-platform.yaml`
- `platforms/k3s/k8s/prometheus-infra/dashboard-logs-vault.yaml`

## Dashboard Scope

| Dashboard | Key Queries |
|-----------|------------|
| ArgoCD | Sync errors, app degraded events, health check failures |
| Temporal | Workflow panics, activity failures, worker errors |
| fourdogs | App error logs, API request errors, DB connection issues |
| Platform | Traefik access errors, ESO sync failures, cert-manager events |
| Vault | Audit deny events, lease failures, auth errors |

## Acceptance Criteria

- [ ] All 5 dashboard ConfigMaps created with `grafana_dashboard: "1"` label
- [ ] All 5 dashboards visible in Grafana after ArgoCD sync
- [ ] Each dashboard loads without datasource errors
- [ ] ArgoCD dashboard shows log entries from `{namespace="argocd"}`
- [ ] Vault dashboard shows entries from `{service="vault", log_type="audit"}` (may be empty until E6)
