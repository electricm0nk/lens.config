# E5-S2 — Alertmanager Routing for Loki Alerts: AlertmanagerConfig CRD

## Story

**As an** operator,  
**I want** Loki ruler alerts to route to Alertmanager,  
**so that** log-based alerts follow the same notification path as Prometheus alerts.

## Context

Alertmanager is already deployed by `prometheus` feature (complete) but has no routes configured. This story adds a `AlertmanagerConfig` CRD for Loki ruler alert routing. Must coordinate with `prometheus-wiring` (finalizeplan-complete) to ensure no config conflict.

**Open item OI-1 (H5):** Before closing this story, check if `prometheus-wiring` has entered dev. If so, coordinate `alertmanagerConfigSelector` in kube-prometheus-stack with the `prometheus-wiring` implementer. If not yet in dev, a minimal catch-all route is acceptable for now with a migration note for when `prometheus-wiring` ships.

**Pre-reqs:** E5-S1 (ruler rules active), Alertmanager running (prometheus feature, complete).

## Files to Create/Modify

- `platforms/k3s/k8s/monitoring/loki-alertmanager-config.yaml` (new `AlertmanagerConfig` CRD)
- Possibly `platforms/k3s/helm/kube-prometheus-stack/values.yaml` (add `alertmanagerConfigSelector` if not set)

## Key Config

```yaml
# loki-alertmanager-config.yaml
apiVersion: monitoring.coreos.com/v1alpha1
kind: AlertmanagerConfig
metadata:
  name: loki-alerts
  namespace: monitoring
  labels:
    alertmanagerConfig: loki
spec:
  route:
    receiver: loki-catch-all
    matchers:
      - name: alertname
        matchType: "=~"
        value: "CrashLoopDetected|OOMKillDetected|VaultAuditDeny|TemporalWorkflowPanic|ArgoCDSyncError"
  receivers:
    - name: loki-catch-all
      # Configure actual receiver when prometheus-wiring routing baseline is established
      # For now: null receiver (silences alert without notification — tracks firing state)
```

**Check `alertmanagerConfigSelector`:**
```bash
kubectl get alertmanager -n monitoring -o yaml | grep -A5 alertmanagerConfigSelector
```

If selector is empty or not set, update `kube-prometheus-stack/values.yaml`:
```yaml
alertmanager:
  alertmanagerSpec:
    alertmanagerConfigSelector:
      matchLabels:
        alertmanagerConfig: loki   # or a broader selector
```
