# E1-S3 — Loki Dev Deployment: Helm Values + ArgoCD App (Upgrade Staging)

## Story

**As an** operator,  
**I want** a loki-dev instance in `monitoring-dev` with local-path PVC,  
**so that** I can safely test Loki chart upgrades against a non-production instance before applying to prod.

## Context

`loki-dev` is NOT a real log collection target. It exists as an upgrade staging environment — the same purpose as `prometheus-dev`. The dev Alloy does not exist; loki-dev is populated only via synthetic data injection during upgrade validation. It uses a 20Gi `local-path` PVC (no S3 credentials needed).

**Relevant architecture sections:**
- `architecture.md` §Phase 1 Step 4 — Loki Dev
- `architecture.md` §GitOps Delivery Pattern §New ArgoCD Applications

## Implementation Tasks

### Task 1: Create Loki Dev Helm Values

Create `platforms/k3s/helm/loki/values-dev.yaml`:

```yaml
loki:
  commonConfig:
    replication_factor: 1
  auth_enabled: false

  storage:
    type: filesystem   # local-path PVC — no S3 for dev

  schemaConfig:
    configs:
      - from: "2026-04-22"
        store: tsdb
        object_store: filesystem
        schema: v13
        index:
          prefix: index_
          period: 24h

  compactor:
    retention_enabled: false   # no retention enforcement for dev

  limits_config:
    retention_period: 168h   # 7d — unenforced; just a config placeholder
    ingestion_rate_mb: 8
    ingestion_burst_size_mb: 16
    max_streams_per_user: 0

singleBinary:
  replicas: 1
  persistence:
    enabled: true
    size: 20Gi
    storageClass: local-path
  resources:
    requests:
      cpu: 50m
      memory: 128Mi
    limits:
      cpu: 200m
      memory: 256Mi
```

### Task 2: Create ArgoCD App Manifest

Create `platforms/k3s/argocd/apps/loki-dev.yaml`:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: loki-dev
  namespace: argocd
  annotations:
    argocd.argoproj.io/sync-wave: "3"
spec:
  project: terminus-infra
  sources:
    - repoURL: https://grafana.github.io/helm-charts
      chart: loki
      targetRevision: "6.*"
      helm:
        valueFiles:
          - $values/platforms/k3s/helm/loki/values-dev.yaml
    - repoURL: https://github.com/electricm0nk/terminus.infra.git
      targetRevision: HEAD
      ref: values
  destination:
    server: https://kubernetes.default.svc
    namespace: monitoring-dev
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=false
```

### Task 3: Verify

```bash
kubectl get pods -n monitoring-dev -l app.kubernetes.io/name=loki -w
kubectl get pvc -n monitoring-dev

# Quick push test for upgrade staging validation
kubectl run -n monitoring-dev loki-dev-test --image=curlimages/curl --restart=Never --rm -it -- \
  curl -XPOST -H "Content-Type: application/json" \
  -d '{"streams":[{"stream":{"job":"dev-test"},"values":[["'$(date +%s%N)'","dev test"]]}]}' \
  http://loki-dev.monitoring-dev.svc.cluster.local:3100/loki/api/v1/push
```
